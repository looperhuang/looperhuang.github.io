迭代與遞迴

發表於 2024-11-27 更新於 2025-04-29 分類於 基礎

迭代(iteration)是一種重複執行任務的控制結構
在一定的條件下重複執行，直到條件不再滿足

  • for
    最常見的迭代形式
int ForLoop(int n)
{
    int result = 0;
    for (int i = 1; i <= n; i--)
    {
        result += i;
    }
    return result;
}
  • while
    while 跟 for 類似，但是 while 會比較好控制條件的設定與變化
    for 的程式碼較簡潔、while 的迴圈控制較靈活
int WhileLoop(int n)
{
    int result = 0;
    int i = 1; //初始條件
    while (i <= n)
    {
        result += i;
        //更新條件
        i +=1;
        i *=2;
    }
    return result;
}
  • 巢狀
    迴圈包迴圈
string NestForLoop(int n)
{
    var result = new StringBuilder();
    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            result.Append($"({i},{j})");
        }
    }
    return result.ToString();
}

遞迴

遞迴(recursion)函式呼叫自己本身
主要三要素

  1. 終止條件 : 什麼時候結束往下層傳遞
  2. 遞迴呼叫 : 將參數往下層傳遞給自己
  3. 結果返回 : 回傳結果給上層
int Recur(int n)
{
    // 終止條件
    if (n == 1) return 1;
    // 遞迴呼叫
    int res = Recur(n - 1);
    // 返回結果
    return n + res;
}

由於遞迴每次都要呼叫重新產生本身函式，會有額外的資源開銷
所以遞迴的次數是有限的，太多記憶體會爆炸，效率也會比迭代來的低

比較

迭代 遞迴
效率 較高，沒有函式產生開銷 每次遞迴呼叫都有產生開銷
記憶體 通常固定大小 根據遞迴函式產生占用大量記憶體
適用 簡單任務、直觀、可讀佳 複雜任務分解(樹、圖、分治、回溯等)、結構明確
