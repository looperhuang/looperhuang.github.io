LINQ ( Language Integrated Query)

微軟開發的技術具有查詢資料的程式語言
讓工程師能夠使用原生 C#對資料進行處理
且相同語法可以查詢不同類型的資料，降低學習成本
擴充類型 :

  • LINQ to Objects
  • LINQ to SQL
  • LINQ to XML
  • …etc

LINQ 查詢的兩種語法

  1. 查詢表達式 ( Query Expression Syntax )
  • 類似 SQL 的語法 : 使用 select 、 from 、where 等關鍵字
  • 較容易理解
  • 結構固定 : 必須 from 起始，select 或 group by 結尾
static void Main()
{
    // Data source.
    int[] scores = new int[] { 90, 71, 82, 93, 75, 82 };

    // Query Expression.
    IEnumerable<int> scoreQuery = //query variable
        from score in scores //required
        where score > 80 // optional
        orderby score descending // optional
        select score; //must end with select or group

    // Execute the query to produce the results
    foreach (var testScore in scoreQuery)
    {
        Console.WriteLine(testScore);
    }

    // Output: 93 90 82 82
}
  1. 流利語法（Fluent Syntax）
  • 靈活性高 : 可以方便地鏈結調用多個方法
  • Lambda 表達式 : 代碼簡潔
  • 沒有固定結構 : 沒有特定的開始或結束語句
static void Main()
{
    // Data source.
    int[] scores = new int[] { 90, 71, 82, 93, 75, 82 };

    // Query Expression.
    var scoreQuery = scores
        .Where(score => score > 80)
        .OrderByDescending(score => score);

    // Execute the query to produce the results
    foreach (var testScore in scoreQuery)
    {
        Console.WriteLine(testScore);
    }

    // Output: 93 90 82 82
}
特徵 查詢表達式 流利語法
風格 類似 sql 方法鏈結
可讀性 高 中
結構要求 from 開頭、select 或 group by 結尾 無特定要求
靈活性 某些情況下較低 更高的靈活性，尤其是動態

時間複雜度

發表於 2024-11-27 更新於 2025-04-25 分類於 基礎

概述

時間複雜度代表的是演算法隨著資料量變大時的增長趨勢
數學符號大O記號(big-O notation)

常見型別

  • 常數階 O(1)
int Constant(int n)
{
    int count = 0;
    // 迴圈次數與 n 無關
    int size = 10000;
    for (int i = 0; i < size; i++)
    {
        count++;
    }
    return count;
}
  • 線性階 O(n)
int Linear(int n)
{
    int count = 0;
    // 迴圈次數 n 次
    for (int i = 0; i < n; i++)
    {
        count++;
    }
    return count;
}
  • 平方階 O(n2n^2n2)
int Quadratic(int n)
{
    int count = 0;
    // 迴圈次數 n*n 次
    for (int i = 0; i < n; i++)
    {
        for (int j = 0; j < n; j++)
        {
            count++;
        }
    }
    return count;
}
  • 指數階 O(2n2^n2n)
int Exponential(int n)
{
    int count = 0;
    int bas = 1;
    for (int i = 0; i < n; i++)
    {
        //迴圈次數為2^n次
        for (int j = 0; j < bas; j++)
        {
            count++;
        }
        bas *= 2;
    }
    return count;
}
  • 對數階 O(logn)
int Logarithmic(int n)
{
    int count = 0;
    while (n > 1) {
        n /= 2;
        count++;
    }
    return count;
}
  • 線性對數階 O(nlogn)
int LinearLogRecur(int n)
{
    if (n <= 1) return 1;
    int count = LinearLogRecur(n / 2);
    for (int i = 0; i < n; i++)
    {
        count++;
    }
    return count;
}
  • 階乘階 O(n!)
int FactorialRecur(int n)
{
    if (n == 0) return 1;
    int count = 0;
    // n * (n - 1) * (n - 2) * ... * 2 * 1
    for (int i = 0; i < n; i++)
    {
        count += FactorialRecur(n - 1);
    }
    return count;
}

迭代與遞迴

發表於 2024-11-27 更新於 2025-04-29 分類於 基礎

迭代(iteration)是一種重複執行任務的控制結構
在一定的條件下重複執行，直到條件不再滿足

  • for
    最常見的迭代形式
int ForLoop(int n)
{
    int result = 0;
    for (int i = 1; i <= n; i--)
    {
        result += i;
    }
    return result;
}
  • while
    while 跟 for 類似，但是 while 會比較好控制條件的設定與變化
    for 的程式碼較簡潔、while 的迴圈控制較靈活
int WhileLoop(int n)
{
    int result = 0;
    int i = 1; //初始條件
    while (i <= n)
    {
        result += i;
        //更新條件
        i +=1;
        i *=2;
    }
    return result;
}
  • 巢狀
    迴圈包迴圈
string NestForLoop(int n)
{
    var result = new StringBuilder();
    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            result.Append($"({i},{j})");
        }
    }
    return result.ToString();
}

遞迴

遞迴(recursion)函式呼叫自己本身
主要三要素

  1. 終止條件 : 什麼時候結束往下層傳遞
  2. 遞迴呼叫 : 將參數往下層傳遞給自己
  3. 結果返回 : 回傳結果給上層
int Recur(int n)
{
    // 終止條件
    if (n == 1) return 1;
    // 遞迴呼叫
    int res = Recur(n - 1);
    // 返回結果
    return n + res;
}

由於遞迴每次都要呼叫重新產生本身函式，會有額外的資源開銷
所以遞迴的次數是有限的，太多記憶體會爆炸，效率也會比迭代來的低

比較

迭代 遞迴
效率 較高，沒有函式產生開銷 每次遞迴呼叫都有產生開銷
記憶體 通常固定大小 根據遞迴函式產生占用大量記憶體
適用 簡單任務、直觀、可讀佳 複雜任務分解(樹、圖、分治、回溯等)、結構明確

C# 資料快取

發表於 2024-11-20 更新於 2025-04-25 分類於 後端

概述

有時候一些大型資料且異動少的資料
每次使用就要從資料重新撈取就很浪費效能資源
這個時候就是快取出場的時候

Install

dotnet add package Microsoft.Extensions.Caching.Memory

Extensions.cs

public static void AddExtension(this IServiceCollection services)
{
    services.AddMemoryCache();
    // services.AddScoped<CacheHelper>(); //如果有使用整合工具類
}

Usage

[Route("api/[controller]")]
 [ApiController]
 public class DataController : ControllerBase
 {
     private readonly DataBaseService dataBaseService;
     private readonly IMemoryCache memoryCache;

     public DataController(DataBaseService dataBaseService, IMemoryCache memoryCache)
     {
         this.memoryCache = memoryCache;
         this.dataBaseService = dataBaseService;
     }

     [HttpGet("/search")]
     public async Task<IActionResult> Search(string keyword)
     {
        // 從記憶體快取查看是否存在，沒再去資料庫撈資料並儲存快取
         if (memoryCache.TryGetValue("MainDataCache", out List<Prs006> dataList))
         {
             var result = dataList?
                 .Where(data => data.Ordnme?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true
                                || data.Scitn?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true
                                || data.Keyword?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true)
                 .ToList() ?? new List<Prs006>();

             return Ok(result);
         }
         else
         {
             var dbData = await dataBaseService.GetDatas();
             //寫入記憶體儲存，一天後自動過期
             memoryCache.Set("MainDataCache", dbData, TimeSpan.FromDays(1));

             var result = dbData?
                 .Where(data => data.Ordnme?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true
                                || data.Scitn?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true
                                || data.Keyword?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true)
                 .ToList() ?? new List<Prs006>();

             return Ok(result);
         }
     }

CacheHelper.cs(Optional)

public class CacheHelper
{
    private readonly IMemoryCache cache;

    public CacheHelper(IMemoryCache cache)
    {
        this.cache = cache;
    }

    public void Set<T>(string key, T value, int expire = 300)
    {
        var options = new MemoryCacheEntryOptions
        {
            AbsoluteExpirationRelativeToNow = TimeSpan.FromSeconds(expire),
        };
        cache.Set(key, value, options);
    }

    public T? Get<T>(string key)
    {
        cache.TryGetValue<T>(key, out var value);
        return value;
    }

    public void Remove(string key)
    {
        cache.Remove(key);
    }

    public void Update<T>(string key, T value, int expire)
    {
        cache.Remove(key);
        Set<T>(key, value, expire);
    }

    public bool Contains(string key)
    {
        return cache.TryGetValue(key, out _);
    }
}

C# 使用 curl.exe 呼叫api

發表於 2024-11-19 更新於 2025-04-25 分類於 後端

概述

有時候開發軟體的版本過舊，導致介接 API 會異常困難
學長說可以用 curl 來呼叫 api，所以就來看看怎麼用

DownLoad curl.exe

https://curl.se/download.html#Win64

CurlHelper.cs

public class CurlHelper
    {
        // curl.exe 路徑
        private static readonly string CurlPath = $"C:\\CURLDEMO\\EXE\\CURL.EXE";

        /// <summary>
        /// Get 請求
        /// </summary>
        /// <param name="url"></param>
        /// <param name="headers"></param>
        /// <returns></returns>
        public static string GetRequest(string url, Dictionary<string, string> headers = null)
        {
            string arugments = $"-X GET \"{url}\"";
            if (headers != null)
            {
                foreach (var header in headers)
                {
                    arugments += $" -H \"{header.Key}: {header.Value}\"";
                }
            }
            return ExecuteCurlCommand(arugments);
        }

        /// <summary>
        /// POST 請求(JSON資料)
        /// </summary>
        /// <param name="url"></param>
        /// <param name="jsonData"></param>
        /// <param name="headers"></param>
        /// <returns></returns>
        public static string PostRequest(string url, string jsonData, Dictionary<string, string> headers = null)
        {
            string arguments = $"-X POST -H \"Content-Type: application/json\" -d \"{jsonData}\" \"{url}\"";
            if (headers != null)
            {
                foreach (var header in headers)
                {
                    arguments += $" -H \"{header.Key}: {header.Value}\"";
                }
            }
            return ExecuteCurlCommand(arguments);
        }

        /// <summary>
        /// POST 請求(表單資料)
        /// </summary>
        /// <param name="url"></param>
        /// <param name="formData"></param>
        /// <param name="headers"></param>
        /// <returns></returns>
        public static string PostFormRequest(string url, Dictionary<string, string> formData, Dictionary<string, string> headers = null)
        {
            string arguments = "-X POST";
            foreach (var item in formData)
            {
                arguments += $" -F \"{item.Key}={item.Value}\"";
            }
            arguments += $" \"{url}\"";
            if (headers != null)
            {
                foreach (var header in headers)
                {
                    arguments += $" -H \"{header.Key}: {header.Value}\"";
                }
            }
            return ExecuteCurlCommand(arguments);
        }


        /// <summary>
        /// 執行curl命令返回輸出
        /// </summary>
        /// <param name="arguments"></param>
        /// <returns></returns>
        private static string ExecuteCurlCommand(string arguments)
        {
            var startInfo = new ProcessStartInfo
            {
                FileName = CurlPath,
                Arguments = arguments,
                RedirectStandardOutput = true,
                UseShellExecute = false,
                CreateNoWindow = true,
            };
            using (var p = Process.Start(startInfo))
            {
                using (var reader = p.StandardOutput)
                {
                    return reader.ReadToEnd();
                }
            }
        }
    }

Usage

string url = "apiurl...";
var headers = new Dictionary<string, string>
{
      {"Authorization", "Bearer your_api_key"}
};
string response = CurlHelper.GetRequest(url, headers);
Console.WriteLine("Get Response: " + response);

C# 使用IdGen產生Id(雪花算法)

發表於 2024-11-19 更新於 2025-04-25 分類於 後端

雪花算法

概述

公司的資料表常常使用很奇怪的欄位來去當識別鍵(例如時間戳)，既然都用時間戳了不如用雪花算法產 ID 當 PK
雪花算法是 Twitter 開源的生成分散式全域產生 ID 的演算法

格式

雪花 ID 有 64 位元

  • 1 位為符號位始終是 0
  • 41 位為時間戳，精度毫秒
  • 10 位可為(10 位機器 ID) 或 (5 位數據中心 ID + 5 位機器 ID)，可識別 1024 台機器
  • 12 位為序列號 => 代表每毫秒可以有 4096 個 ID

優缺點

優點

  1. ID 包含時間戳，有序可排序
  2. 簡單速度快

缺點

  1. 依賴系統時間，如果系統時間回退或跳躍，會產生不唯一 ID
  2. 自己手動分配機器 ID，分配不當，產生不唯一 ID

Install

dotnet add package IdGen

DI 注入(Optionl)

dotnet add package IdGen.DependencyInjection

Usage

一般

var id = generator.CreateId();

Controller.cs

public class TasController : ControllerBase
{

    private readonly ISchedulerFactory schedulerFactory;
    private readonly IdGenerator idGenerator;

    public TasController(ISchedulerFactory schedulerFactory,
                         IdGenerator idGenerator)
    {
        this.schedulerFactory = schedulerFactory;
        this.idGenerator = idGenerator;
    }

    [HttpPost("[action]")]
    public async Task<IActionResult> ScheduleTasJob(TasJobRequest request)
    {
        // 使用 SchedulerFactory 取得 Scheduler
        var sch = await schedulerFactory.GetScheduler();
       // 雪花算法產Id
        var jobid = idGenerator.CreateId();
        var jobKey = new JobKey($"TasJob-{jobid}", "TasJobs");
        var triggerKey = new TriggerKey($"TasJobTrigger-{jobid}", "TasJobs");

        // 建立Job
        var job = JobBuilder.Create<SendTasJob>()
            .WithIdentity(jobKey)
             //key value 如果有開啟UseProperties 只能有string型別
            .UsingJobData("tasid", request.TasId)
            .UsingJobData("welcomeText", request.WelcomeText)
            .StoreDurably() // 使任務持久化
            .Build();

        // 設定Trigger
        ITrigger trigger = TriggerBuilder.Create()
            .WithIdentity(triggerKey)
            .StartAt(request.ExecuteAt ?? DateTime.UtcNow) // 指定執行時間
            .WithSimpleSchedule(x => x
                .WithMisfireHandlingInstructionFireNow() // 當排程錯過執行時間時立即執行
                .WithRepeatCount(0) //只執行一次
                )
            .Build();

        try
        {
           await sch.ScheduleJob(job, trigger);
           Console.WriteLine($"Scheduled job successfully with JobKey: {jobKey.Name} and TriggerKey: {triggerKey.Name}");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"Failed to schedule job with JobKey: {jobKey.Name} and TriggerKey: {triggerKey.Name}", ex);
            return StatusCode(500, "Failed to schedule job");
        }

        return Ok($"Scheduled job successfully with Job: {jobKey.Name} and Trigger: {triggerKey.Name}");
    }
}

AddQuartzServer vs AddQuartzHostedService

AddQuartzServer 會多一個 Health Check 檢查 scheduler
Source Code 如下

using System;

using Microsoft.Extensions.DependencyInjection;

#if SUPPORTS_HEALTH_CHECKS
using Quartz.AspNetCore.HealthChecks;
#endif

namespace Quartz
{
    public static class QuartzServiceCollectionExtensions
    {
        public static IServiceCollection AddQuartzServer(
            this IServiceCollection services,
            Action<QuartzHostedServiceOptions>? configure = null)
        {
            #if SUPPORTS_HEALTH_CHECKS
            services
                .AddHealthChecks()
                .AddTypeActivatedCheck<QuartzHealthCheck>("quartz-scheduler");
            #endif

            return services.AddQuartzHostedService(configure);
        }
    }
}
