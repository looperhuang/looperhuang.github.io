🦀 Rust The Slice Type

發表於 2025-05-21 分類於 rust

📌 什麼是 Slice？

在 Rust 中，Slice 是一種 不具所有權的資料引用。
它允許你引用一段連續的元素（如字串或陣列）而非整個集合，實現更靈活與安全的資料處理。

📚 基本語法

let s = String::from("hello world");
let hello = &s[0..5];  // "hello"（索引 0 到 4）
let world = &s[6..11]; // "world"（索引 6 到 10）

📏 簡化語法

Rust 提供更簡潔的切片語法：

  • &s[..5] 等同於 &s[0..5]
  • &s[6..] 等同於 &s[6..s.len()]
  • &s[..] 代表整個字串的切片

🔐 Slice 的安全性

Rust 會在執行時檢查索引是否越界，避免非法記憶體存取。

let s = String::from("hello");
let slice = &s[0..6]; // ⚠️ 錯誤！超出索引範圍

🧠 使用範例：尋找第一個單字的結尾

fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]
}

此函式尋找第一個空格前的字串切片。

❗ 為何使用 Slice？

可以避免資料與索引失效之間的錯誤關聯：

fn main() {
    let mut s = String::from("hello world");
    let word = first_word(&s);
    s.clear(); // 清空整個字串
    println!("the first word is: {word}"); // ❌ 錯誤！使用了已失效的 slice
}

Rust 的 借用檢查器（borrow checker） 將禁止這種情況，強化記憶體安全。

🧩 使用 &str 取代 &String（參數泛化）

❌ 原本版本：只接受 &String


fn first_word(s: &String) -> &str {
    // ...略
}

限制只能傳入 &String，無法處理字串字面值（例如 "hello world"）。

✅ 改寫版本：接受更通用的 &str

fn first_word(s: &str) -> &str {
    // ...略
}

這樣的版本更通用，支援所有字串切片，包括：

  • 字串字面值："hello world"
  • String 型別：String::from("...") 可自動轉為 &str

🧪 使用範例

fn main() {
    let s1 = String::from("hello world");
    let s2 = "hello world"; // 字串字面值（&'static str）

    // 都可以傳入 &str 函式
    let word1 = first_word(&s1); // OK，自動轉型為 &str
    let word2 = first_word(s2);  // OK，已是 &str

    println!("word1: {}", word1);
    println!("word2: {}", word2);
}

📌 其他類型的切片

  • 陣列
let a = [1, 2, 3, 4, 5];
let slice = &a[1..3]; // [2, 3]

✅ 小結

概念 說明
Slice 不擁有資料的部分引用
語法 &s[start..end]
用途 引用部分資料，如字串的一部分
安全性 編譯器與執行時會檢查索引範圍
借用規則整合 與所有權和借用機制緊密整合，防止錯誤

🦀 Rust References and Borrowing

發表於 2025-05-16 分類於 rust

🧠 References and Borrowing

在 Rust 中，ownership（所有權） 是記憶體安全的基石，而「reference（引用）」與「borrowing（借用）」則允許我們不轉移所有權的情況下存取資料。

📌 為什麼要使用 References?

Rust 不允許在沒有 ownership 的情況下自由存取資料。
但有時我們只需要讀取資料而不需要修改，這時可以使用 reference（&） 來借用資料：

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);
    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
  • &s1：建立對 s1 的參考（reference）
  • s 是 &String 類型，代表它是引用
  • s1 的 ownership 沒有被移走，可以在之後繼續使用

⚠️ 借用（Borrowing）規則

Rust 有嚴格的借用規則，確保不會發生資料競爭（data race）：

  1. 同一時間只能有一個可變參考 &mut T，或
  2. 任意數量的不可變參考 &T
  3. 引用在使用前必須有效（valid）

❌ 錯誤範例：同時存在可變與不可變參考

let mut s = String::from("hello");

let r1 = &s;
let r2 = &s;
let r3 = &mut s; // ❌ 錯誤：不可在不可變引用存在時建立可變引用

println!("{}, {}, and {}", r1, r2, r3);

✅ 正確範例：分開生命週期

let mut s = String::from("hello");

let r1 = &s;
let r2 = &s;
println!("{}, {}", r1, r2); // 先使用不可變參考

let r3 = &mut s; // ✅ 合法：因為 r1, r2 已經使用完（生命週期已結束）
println!("{}", r3);

✍️ Mutable References（可變引用）

當我們需要修改變數的值，但又不想移轉 ownership 時，可以使用 可變引用（&mut）。

✅ 可變引用範例：

fn main() {
    let mut s = String::from("hello");

    change(&mut s); // 傳入可變引用，函式內可修改 s 的值

    println!("{}", s); // output: hello, world!
}

fn change(some_string: &mut String) {
    some_string.push_str(", world!");
}
  • &mut s 表示建立一個指向 s 的可變引用。
  • change 函數內可以修改傳入的 String。
  • 必須先宣告變數為 mut，才能建立可變借用。

⚠️ 可變借用的限制

  1. 不可同時有多個可變借用
  2. 不可同時存在可變與不可變借用

✅ 技巧：可變引用的「作用域結束」讓你重新借用

let mut s = String::from("hello");

{
    let r1 = &mut s;
} // r1 在這裡離開作用域

let r2 = &mut s; // ✅ 合法，因為 r1 已結束作用域
r2.push_str(" world");

🧹 懸空引用（Dangling Reference）

在有指針的語言，很容易因為釋放記憶體，但指針還保留，導致指針指向一個沒有值的記憶體位置，我們稱之為懸空指針(dangling pointer)。
在 Rust 會避免「懸空引用（dangling reference）」的產生：

❌ 錯誤範例

fn dangle() -> &String {
    let s = String::from("hello");
    &s // ❌ 錯誤：s 在此離開作用域，會釋放記憶體
}

✅ 正確範例

fn no_dangle() -> String {
    let s = String::from("hello");
    s
}

📝 小結

概念 說明
不可變借用 &T - 可建立多個同時存在的不可變引用
- 無法修改所參考的值
可變借用 &mut T - 一次只能存在一個可變引用
- 可以修改所參考的值
混合借用禁止 - 在可變借用存在時，不可同時存在任何不可變借用（反之亦然）
借用作用域 - 借用的作用域由變數使用範圍決定
- 借用結束後才能建立新的借用
懸空引用禁止 - Rust 編譯器會防止返回指向已釋放變數的引用（避免懸空指標 / dangling reference）
目的與優點 - 保證程式安全、無資料競爭（data race）
- 在編譯期間就避免常見錯誤

🦀 Rust 所有權 Ownership 基礎概念

發表於 2025-05-15 分類於 rust

Rust 不像 C# 使用垃圾回收器（GC）來自動管理記憶體，也不像 C/C++ 那樣需要手動釋放記憶體（如 free()）。
Rust 採用的是一套獨特的所有權（Ownership）系統，在編譯階段就保證記憶體安全與釋放。

📖 本筆記整理自 Rust 官方書籍：Rust Book §4.1 - What is Ownership?

🧠 所有權 Ownership 是什麼？

Rust 使用 所有權（Ownership） 來管理記憶體的分配與釋放，這是語言的核心機制之一。

三大原則：

  1. 每個值都有一個所有者（Owner）
  2. 同一時間只能有一個所有者
  3. 當所有者離開作用域，值就會被釋放（Drop）

✏️ 所有權示範

fn main() {
    let s = String::from("hello"); // s 擁有這個字串
    println!("{}", s);
} // 這裡 s 離開作用域，自動呼叫 drop，記憶體被釋放

♻️ 所有權轉移（Move）

當你把變數賦值給另一個變數時，如果它是 heap 分配的資料（像 String），Rust 會直接移轉所有權。

這樣做是為了避免雙重釋放（double free）的錯誤。

fn main() {
    let s1 = String::from("hello");
    let s2 = s1;

    // println!("{}", s1); // ❌ 編譯錯誤：s1 不再有效
    println!("{}", s2); // ✅
}

🧬 深層複製：Clone

如果你真的需要「複製一份資料」而不是移轉所有權，可以使用 .clone() 方法。

這會產生資料的 深層複製（deep copy），適用於像 String, Vec 等 heap 型別。

fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone(); // 複製出一份新的 String

    println!("s1 = {}, s2 = {}", s1, s2); // ✅ 兩個都可以使用
}

🆚 Move vs Clone 的差別

操作 行為 所有權變化 效能成本
let b = a; 預設會 Move a 無效 快（無複製）
let b = a.clone(); 深層複製 a 保持有效 需要額外成本（複製記憶體）

📝 建議只有在真的需要保留原資料時才使用 .clone()，否則會帶來效能上的開銷。

📋 複製 Copy

某些型別（如整數、布林值、浮點數、字元、tuple（包含 primitive））支援 Copy trait，不會移動所有權。

fn main() {
    let x = 5;
    let y = x;

    println!("x = {}, y = {}", x, y); // ✅ OK，因為 i32 是 Copy
}

哪些型別是 Copy？

  • 所有 primitive 型別（如 i32, f64, bool, char）
  • 不包含 heap 分配的 tuple，例如 (i32, bool) 是 Copy，但 (String, i32) 不是。

🧾 函式參數與所有權

呼叫函式也會將值「移動」或「複製」進去參數：

fn main() {
    let s = String::from("hello");
    takes_ownership(s); // s 被移走
    // println!("{}", s); // ❌ 錯誤

    let x = 5;
    makes_copy(x); // x 被複製，仍可用
    println!("{}", x);
}

fn takes_ownership(some_string: String) {
    println!("{}", some_string);
}

fn makes_copy(some_int: i32) {
    println!("{}", some_int);
}

🔁 返回所有權

函式可以回傳值，將所有權還回來：

fn main() {
    let s1 = gives_ownership(); // 所有權轉給 s1
    let s2 = String::from("world");
    let s3 = takes_and_gives_back(s2); // s2 被移走後傳回給 s3
}

fn gives_ownership() -> String {
    String::from("hello")
}

fn takes_and_gives_back(s: String) -> String {
    s
}

📝 小結

操作 是否移動所有權 範例
賦值給其他變數 ✅（非 Copy） let s2 = s1;
傳入函式參數 ✅（非 Copy） takes_ownership(s)
回傳值 ✅ let s = func();
Copy 類型 ❌ i32, bool, char, tuple
Heap 型別 ✅ String, Vec

🦀 Rust 基礎教學：流程控制（Control Flow）

發表於 2025-05-15 分類於 rust

在這篇學習筆記中，我們會介紹 Rust 中常見的流程控制方式，包括：

  • 條件判斷：if
  • 無限迴圈：loop
  • 條件迴圈：while
  • 集合迴圈：for

這些控制結構與其他語言類似，但也有一些語法特色和安全性設計值得注意。

📖 本筆記整理自 Rust 官方書籍：Control Flow

✅ if 條件判斷

Rust 的 if 表達式 不需要括號 包住條件，但主體需用大括號 {} 包起來。

fn main() {
    let number = 7;

    if number < 5 {
        println!("小於 5");
    } else if number == 7 {
        println!("是 7");
    } else {
        println!("大於等於 5 且不等於 7");
    }
}
  • 條件必須是布林值（不能像 C 語言、js 那樣用整數當作條件）。
  • 每個區塊需用{}包起來。

💡 if 是一個「表達式」

你可以將 if 當作一個值來賦值給變數。前提是兩個分支的型別必須一致：

fn main() {
    let condition = true;

    let number = if condition {
        5
    } else {
        6
    };

    println!("The number is: {}", number); // 印出 5
}

❗ 如果 if 和 else 的區塊回傳型別不同，會產生編譯錯誤。

🔁 loop：無限迴圈

loop 是 Rust 中最基本的迴圈，會永遠執行直到被手動跳出（通常搭配 break 使用）。

fn main() {
    loop {
        println!("再來一次！");
        break;
    }
}

loop 可以回傳值！

fn main() {
    let result = loop {
        let x = 2 + 2;
        if x == 4 {
            break x * 10; // 回傳值
        }
    };

    println!("結果是: {}", result); // 40
}

💡 使用 break <值> 可以讓 loop 的值被賦給變數。

🌀 while：條件迴圈

while 會根據條件判斷是否繼續執行。

fn main() {
    let mut number = 3;

    while number != 0 {
        println!("{}!", number);
        number -= 1;
    }

    println!("發射！");
}

⚠️ 不建議用 while 走訪集合（因為容易出錯），建議使用 for。

🔁 for：集合迴圈（推薦使用）

for 是走訪集合最安全、最簡潔的方法，避免了索引溢位等問題。

fn main() {
    let a = [10, 20, 30, 40, 50];

    for element in a.iter() {
        println!("元素是: {}", element);
    }
}

搭配範圍語法

你也可以用 for 搭配範圍（range）語法進行倒數：

fn main() {
    for number in (1..=3).rev() {
        println!("{}!", number);
    }
    println!("發射！");
}

// (1..=3) 表示包含 1 到 3，.rev() 反轉順序。

📝 小結

語法 說明
if 條件判斷，不能省略布林值，表達式
loop 永遠執行，直到 break 結束
while 根據條件執行，類似其他語言
for 走訪集合或範圍，最推薦的迴圈寫法

🦀 Rust 基礎教學：函式的使用與運作方式

發表於 2025-05-14 更新於 2025-05-15 分類於 rust

在 Rust 中，函式（Function）是一等公民。
這篇筆記會介紹函式的定義、參數、回傳值，以及函式表達式與語句的差別。

📖 參考自 Rust 官方書籍：How Functions Work

✅ 函式基本結構

Rust 中的函式使用 fn 關鍵字定義，函式名稱慣例使用 snake_case 命名。

fn main() {
    println!("Hello from the main function!");
    another_function();
}

fn another_function() {
    println!("Hello from another function!");
}

執行結果：

Hello from the main function!
Hello from another function!

📥 函式參數（Parameters）

函式可以接受參數，並透過型別註記指定參數型別。
Rust 是靜態型別語言，必須為每個參數指定型別。

fn main() {
    print_number(42);
    print_two_numbers(10, 20);
}

fn print_number(x: i32) {
    println!("The number is: {}", x);
}

fn print_two_numbers(a: i32, b: i32) {
    println!("a = {}, b = {}", a, b);
}

💡 多個參數間用逗號分隔，型別註記不能省略。

🔁 表達式與語句的差異

Rust 的函式主體由「語句（statements）」和「表達式（expressions）」組成。
理解這兩者的差別對寫出正確的函式非常重要。

  • 語句：執行動作，但不回傳值。例如 let x = 5; 是語句。
  • 表達式：會產生值，可以用在變數賦值或函式回傳中。例如 5 + 1 是表達式。
fn main() {
    let y = {
        let x = 3;
        x + 1 // 注意：這裡沒有分號，這是表達式
    };

    println!("The value of y is: {}", y); // y = 4
}

⚠️ 如果在 x + 1; 加了分號，它就變成語句，y 將不會得到任何值，會導致錯誤。

🔚 回傳值（Return Values）

Rust 的函式可以回傳值，透過函式主體中最後一行的表達式實現。
不需要使用 return 關鍵字（雖然也可以使用）。

fn five() -> i32 {
    5 // 回傳值是這個表達式
}

fn plus_one(x: i32) -> i32 {
    x + 1
}

fn main() {
    let x = five();
    let y = plus_one(x);
    println!("x = {}, y = {}", x, y); // x = 5, y = 6
}
  • -> i32 表示回傳一個整數型別。
  • 最後一行不能有分號，否則會變成語句而非回傳值。

✨ 可以使用 return 提早結束函式並指定回傳值，例如 : return x + 1;

📌 小結

概念 重點說明
函式定義 使用 fn 關鍵字，名稱使用 snake_case 命名
函式參數 必須指定型別，格式為 name: type
語句 執行某些動作，不回傳值
表達式 產生值，可以回傳 或用於賦值
回傳值 最後一行為表達式，或使用 return 明確回傳

🦀 Rust 基礎教學：變數、常數與遮蔽（Shadowing）

發表於 2025-05-13 更新於 2025-05-15 分類於 rust

📌 變數的宣告

Rust 使用 let 來宣告變數，預設為不可變（immutable）。如果想讓變數的值可以改變，需要加上 mut。

let a = 456;       // 不可變變數
let mut b = 789;   // 可變變數
println!("a = {}, b = {}", a, b);

// a = 999; // ❌ 錯誤：a 是不可變的
b = 111;   // ✅ 合法：b 是可變的
println!("a = {}, b = {}", a, b);

🔒 常數（const）

常數使用 const 宣告，與 let 不同的是：

  • 必須加上資料別
  • 永遠不可變(不能加 mut)
  • 通常用於全域設定或固定值
  • 命名習慣 SCREAMING_SNAKE_CASE
const MAX_POINTS: u32 = 100_000;
println!("The maximum points are: {}", MAX_POINTS);

⚠️ 常數不能遮蔽（shadow），也不能重新指定

🎭 變數遮蔽（Shadowing）

Rust 允許重新宣告相同名稱的變數，這稱為遮蔽（shadowing）。
遮蔽可以讓你重新指定值甚至改變變數型別，而不用使用 mut。

let c = 10;
let c = "abc"; // 遮蔽了上一個 c，型別從 i32 → &str
println!("The value of c is: {}", c);

與mut有何不同?

let mut x = 5;
x = "hello"; // ❌ 錯誤：型別不能變

let y = 5;
let y = "hello"; // ✅ 合法：因為是新的變數，遮蔽前一個

✅ 使用遮蔽能讓變數在不改變可變性的情況下重新定義或轉換型別。

📝 小結

宣告方式 是否可變 是否可遮蔽 是否需要型別註記 命名風格
let ❌ 預設不可變（可加 mut） ✅ 可以遮蔽 ❌ 可省略 snake_case
const ❌ 永遠不可變 ❌ 不可遮蔽 ✅ 必須註明型別 SCREAMING_SNAKE_CASE

讓ASP.NET Core服務註冊更優雅

發表於 2025-05-13 分類於 後端

前言

隨著應用程式變得越來越龐大，需要註冊的服務也隨之增加，Program.cs 中的 builder.Services 區塊可能會變得冗長且難以維護。
這時候，IServiceCollection 的擴充方法便能派上用場，幫助我們整理程式碼，使其更加清晰和易於管理。

本文範例基於 .NET 6 或更新版本（如 .NET 8）

什麼是 IServiceCollection 擴充方法？

簡單來說，擴充方法允許我們向現有的型別(IServiceCollection介面)添加新的方法，而無需修改原始型別的程式碼。
這是一個強大的 C#語言特性，可以幫助我們編寫更具表達力和可重用的程式碼。

為什麼要使用 IServiceCollection 擴充方法？

使用IServiceCollection擴充方法帶來了諸多好處

  • 提升可讀性：將特定功能的服務註冊邏輯封裝在擴充方法中，可使 Program.cs 更加簡潔，一眼看出應用程式的核心服務。
  • 增加可重用性：若多個專案或模組需註冊相同的服務集合，可將邏輯提取成擴充方法，便於重複使用。
  • 更好的組織結構：可依照模組或類型，將註冊邏輯組織到不同擴充類別中。
  • 簡化複雜的註冊邏輯：封裝需額外配置或條件註冊的服務，讓主要註冊區塊更乾淨。

如何建立和使用 IServiceCollection 擴充方法？

建立擴充方法很簡單，只需定義一個靜態類別，並撰寫靜態方法，第一個參數用 this 修飾 IServiceCollection。

1️⃣ 註冊自定義服務

假設我們有一個 MyAwesomeService 實作 IMyAwesomeService，我們希望註冊為單例：

// Extensions/ServiceCollectionExtensions.cs
using Microsoft.Extensions.DependencyInjection;

public static class ServiceCollectionExtensions
{
    public static IServiceCollection AddMyAwesomeService(this IServiceCollection services)
    {
        return services.AddSingleton<IMyAwesomeService, MyAwesomeService>();
    }
}

public interface IMyAwesomeService
{
    void DoSomethingAwesome();
}

public class MyAwesomeService : IMyAwesomeService
{
    public void DoSomethingAwesome()
    {
        Console.WriteLine("執行一些很棒的操作！");
    }
}

然後在 Program.cs 中使用

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddMyAwesomeService();
// 其他服務註冊...

var app = builder.Build();
app.Run();

2️⃣ 註冊多個相關服務

我們可以一次註冊一組資料存取服務：

// Extensions/DataAccessServiceExtensions.cs
public static class DataAccessServiceExtensions
{
    public static IServiceCollection AddDataAccessServices(this IServiceCollection services, string connectionString)
    {
        services.AddScoped<IDataContext>(provider => new DataContext(connectionString));
        services.AddScoped<IUserRepository, UserRepository>();
        services.AddScoped<IProductRepository, ProductRepository>();
        return services;
    }
}

在 Program.cs 中使用：

var builder = WebApplication.CreateBuilder(args);
var connectionString = builder.Configuration.GetConnectionString("DefaultConnection");

builder.Services.AddDataAccessServices(connectionString);

var app = builder.Build();
app.Run();

3️⃣ 根據環境註冊不同的服務

根據開發或生產環境註冊不同的實作類別：

// Extensions/EnvironmentSpecificServiceExtensions.cs
using Microsoft.Extensions.Hosting;

public static class EnvironmentSpecificServiceExtensions
{
    public static IServiceCollection AddEnvironmentSpecificServices(this IServiceCollection services, IHostEnvironment environment)
    {
        if (environment.IsDevelopment())
        {
            services.AddTransient<IEmailService, DevelopmentEmailService>();
        }
        else
        {
            services.AddTransient<IEmailService, ProductionEmailService>();
        }
        return services;
    }
}

public interface IEmailService
{
    void SendEmail(string to, string subject, string body);
}

public class DevelopmentEmailService : IEmailService
{
    public void SendEmail(string to, string subject, string body)
    {
        Console.WriteLine($"[開發環境] 發送郵件至 {to}，標題：{subject}");
    }
}

public class ProductionEmailService : IEmailService
{
    public void SendEmail(string to, string subject, string body)
    {
        Console.WriteLine($"[生產環境] 正在發送郵件至 {to}，標題：{subject}...");
        // 真正的發送程式碼
    }
}

在 Program.cs 中使用：

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddEnvironmentSpecificServices(builder.Environment);

var app = builder.Build();
app.Run();

結語

IServiceCollection 擴充方法是一個強大且實用的工具，能幫助我們更好地組織與管理 ASP.NET Core 中的服務註冊邏輯。
透過封裝服務註冊至擴充方法，可讓 Program.cs 更加清晰、模組化，並提升可維護性與重用性。

Dijkstra 演算法

發表於 2025-05-07 分類於 基礎

🧠 簡述

由荷蘭計算機科學家 Edsger W. Dijkstra 於 1956 年提出

  • 用來尋找單一源點到其他節點的最短路徑
  • 適用於加權圖(所以邊的權重為正)
  • 常用的資料結構 : 優先佇列(Priority Queue) 或 最小堆(Min-Heap)

🧭 步驟

  1. 將所有節點分為兩組：已確定最短路徑（S）與未確定（U）
  2. 初始時，起點在 S，其餘在 U，距離設為 ∞，起點設為 0
  3. 從 U 中選擇目前與起點距離最短的節點加入 S
  4. 更新該節點鄰接點的距離（鬆弛）
  5. 重複步驟 3~4，直到所有節點都加入 S 或無法再更新

🧑‍💻 C# 實作（含範例圖）

以下為對應圖形的 C# 程式實作

static void Main()
{
    int n = 7; // 節點數
    List<Edge>[] graph = new List<Edge>[n];
    for (int i = 0; i < n; i++) graph[i] = new List<Edge>();

    // 建立圖的鄰接表（根據圖像）
    AddEdge(graph, 0, 1, 2);
    AddEdge(graph, 0, 2, 6);
    AddEdge(graph, 1, 3, 5);
    AddEdge(graph, 2, 3, 8);
    AddEdge(graph, 3, 4, 10);
    AddEdge(graph, 3, 5, 15);
    AddEdge(graph, 4, 5, 6);
    AddEdge(graph, 4, 6, 2);
    AddEdge(graph, 5, 6, 6);

    int start = 0;
    int[] dist = Dijkstra(graph, start);

    Console.WriteLine("起點到各節點的最短距離");
    for (int i = 0; i < n; i++)
    {
        Console.WriteLine($"到節點{i}的最短距離 : {dist[i]}");
    }
}

class Edge
{
    public int To, Weight;
    public Edge(int to, int weight)
    {
        To = to; Weight = weight;
    }
}

static void AddEdge(List<Edge>[] graph, int from, int to, int weight)
{
    graph[from].Add(new Edge(to, weight));
    graph[to].Add(new Edge(from, weight)); // 無向圖需要雙向邊
}

static int[] Dijkstra(List<Edge>[] graph, int start)
{
    int n = graph.Length;
    int[] dist = new int[n];
    bool[] visited = new bool[n];

    // 初始化距離與訪問狀態
    for (int i = 0; i < n; i++) dist[i] = int.MaxValue;
    dist[start] = 0;

    // 使用NET6+ 的 PriorityQueue<TElement, TPriority>
    // 舊版要自己實作 MinHeap
    // 使用優先佇列處理節點，依距離最小優先處理
    var pq = new PriorityQueue<int, int>();
    pq.Enqueue(start, 0);

    while (pq.Count > 0)
    {
        int u = pq.Dequeue();

        // 確認是否處理過節點，並更新註記
        if (visited[u]) continue;
        visited[u] = true;

        // 遍歷所有相鄰邊
        foreach (var edge in graph[u])
        {
            int v = edge.To;
            int weight = edge.Weight;

           // 若從 u 到 v 的新距離更短，則更新 dist 並放入優先佇列
            if (dist[u] + weight < dist[v])
            {
                dist[v] = dist[u] + weight;
                pq.Enqueue(v, dist[v]);
            }
        }
    }

    return dist;
}

WebRTC

發表於 2025-05-01 分類於 後端

簡易記錄一下 WebRTC 的基本概念

基本建立點對點連線的流程

信令 (Signaling)：

  1. 兩個 WebRTC 客户端（Peer A 和 Peer B）透過一個信令伺服器進行初始的資訊交換。這個伺服器不參與實際的媒體傳輸。
  2. Peer A (發起方 - Offerer) 創建一個包含其媒體能力和網路資訊的 SDP Offer。
  3. Peer A 收集其 ICE Candidates（潛在的網路位址）。
  4. Peer A 將 SDP Offer 和 ICE Candidates 透過信令伺服器發送給 Peer B。
  5. Peer B (接收方 - Answerer) 收到 SDP Offer 和 ICE Candidates。
  6. Peer B 設定遠端的 SDP Offer，並創建包含其自身媒體能力和網路資訊的 SDP Answer。
  7. Peer B 也收集其 ICE Candidates。
  8. Peer B 將 SDP Answer 和 ICE Candidates 透過信令伺服器發送回 Peer A。
  9. Peer A 收到 SDP Answer 和 ICE Candidates，並設定遠端的 SDP Answer。
  10. 雙方持續透過信令伺服器交換 ICE Candidates。

ICE 連線建立 (ICE Connection Establishment)：

  1. 每個 Peer 在收到對方的 ICE Candidates 後，會嘗試與這些候選者建立網路連線。
  2. 這個過程會嘗試各種可能的路徑（直接連接、透過 STUN、透過 TURN）。
  3. 當找到一個可用的連線路徑時，ICE 連線建立成功。

媒體和資料傳輸：

  1. 一旦 ICE 連線建立，兩個 Peer 就可以直接透過這個點對點連線傳輸音訊、視訊或任意資料（使用 Data Channels）。

相關名詞簡介：

WebRTC (Web Real-Time Communication)：

一個開放原始碼專案，提供瀏覽器和行動應用程式之間即時音訊、視訊和資料交換的能力，無需外掛程式。(目前只有 js)

Peer (對等方)：

在 WebRTC 通訊中的一個端點，通常是一個瀏覽器或應用程式。

點對點 (Peer-to-Peer, P2P)：

一種網路模型，其中兩個或多個參與者可以直接互相通信，而不需要透過中央伺服器（在媒體和資料傳輸階段）。

信令 (Signaling)：

在 WebRTC 連線建立之前，兩個 Peer 之間交換控制資訊（例如 SDP 和 ICE Candidates）的過程。信令機制不是 WebRTC 本身定義的，需要開發者自行實作。

信令伺服器 (Signaling Server)：

一個中介伺服器，用於在 WebRTC 連線建立的初始階段協助 Peer 發現彼此並交換信令訊息。

SDP (Session Description Protocol)：

一種標準化的文字格式，用於描述多媒體會話的屬性，例如支援的媒體類型、編解碼器、網路位址等。在 WebRTC 中，SDP 用於在連接建立前協商媒體格式。

Offer/Answer 模型 (Offer/Answer Model)：

WebRTC 使用這個模型來協商會話參數。發起方創建一個 Offer SDP，接收方回應一個 Answer SDP。

ICE (Interactive Connectivity Establishment)：

一個用於 NAT 穿透的協定。ICE 的目標是找到兩個 Peer 之間最佳的網路路徑來建立直接連線。

ICE Candidate (ICE 候選者)：

描述一個 Peer 可能的網路位址和連接埠的資訊，用於 ICE 連線建立過程。

STUN (Session Traversal Utilities for NAT) 伺服器：

一種協助位於 NAT 後面的 Peer 發現其公有 IP 位址和連接埠的伺服器。

TURN (Traversal Using Relays around NAT) 伺服器：

一種在直接連線和 STUN 無法工作時，作為中繼伺服器轉發 Peer 之間流量的伺服器。

NAT (Network Address Translator)：

一種網路設備，用於將私有網路中的多個裝置的 IP 位址轉換為一個或少數幾個公有 IP 位址，以便它們可以存取網際網路。NAT 會導致直接的點對點連線變得困難。
