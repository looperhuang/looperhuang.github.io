系統架構

發表於 2025-02-13 更新於 2025-02-14 分類於 linux

BIOS

基本輸入輸出系統(Basic Input/Output System)

主要作用為將系統硬體帶到可以準備交給作業系統的狀態，通常包含下列步驟:

  • 系統初始化
  • 測試記憶體與其他周邊
  • 檢測儲存裝置
  • 在數個儲存裝置之間找出作業系統的起始點

且提供操作介面，供使用者設定系統的低階參數、選擇開機裝置、配置系統資源等等。

日期與時間

BIOS 的主要功能之一是管理主機板內建的硬體時鐘。
作業系統本身也有設定硬體時鐘的能力，可以依靠精準的外部較時參考來源(ex: NTPD 伺服器)來保持硬體時鐘的精準。

磁碟與開機裝置

BIOS 設定工具中其中一項重要功能就是開機裝置的選擇。
大多數 PC 至少支援三種開機媒體

  • 內接式硬碟(IDE 或 SCSI 或 SATA，或三者皆是)
  • 光碟機(同上)
  • 軟碟機

運用/proc 檔案系統

/proc 檔案系統 (核心的運作狀態庫)目錄下的 interrupts、dma、ioports 三個檔案，提供目前系統資源的使用狀況

cat /proc/dma

USB

USB 拓樸邏輯

電腦主機與 USB 裝置之間，是透過集線器(HUB)連接一起，構成一個樹狀結構。
使用lsusb指令可顯示 Linux 系統上所有 USB 裝置的實際連接方式。

USB 控制器

主要分成 4 種類型

  • 開放型主控界面 ( Open Host Controller Interface , OHCI )
  • 通用型主控界面 ( Universal Host Controller Interface , UJCI )
  • 強化型主控界面 ( Enhanced Host Controller Interface , EHCI )
  • 擴充型主控界面 ( eXtented Host Controller Interface , xHCI )
規格 控制器 傳輸速率 向下相容
USB1.1 OHCI、UHCI 12Mbps 否
USB2.0 EHCI 480Mbps 否
USB3.0 xHCI 5Gbps 是

USB 裝置

  • 人機界面裝置 ( Human Interface Device , HID)
    輸入裝置 (滑鼠、鍵盤…等等)
  • 通訊裝置
    數據機
  • 大量儲存裝置
    磁碟機、讀卡機…等
  • 音訊
    揚聲器、音效裝置
  • IrDA
    紅外線裝置
  • 印表機
    印表機以及 USB-to-parallel 轉換器

USB 驅動程式

Linux 核心 2.0.x 以及更早版本，不支援 USB

主要分成三大類

  1. 主控器驅動程式
  2. 通類驅動程式
  3. 雜項裝置驅動程式

USB 熱插拔

模組化的 USB 驅動程式，是藉由核心對於/sbin/hotplug 的通用支援機制來載入

樹狀資料

發表於 2025-02-05 分類於 資料結構

樹狀結構介紹

  • Node : 樹的基本單位，紀錄節點資料與子節點有哪些
  • Root Node : 根節點，樹的最高節點
  • Leaf Node : 葉節點，沒有子節點的節點
  • Edge : 連接兩個節點的線，表示節點之間的關係
  • Parent Node : 父節點(上層節點)，一個節點只會有一個父節點
  • Child Node : 子節點，一個節點可以連結多個子節點
  • Sibling Node : 兄弟節點，有相同父節點的節點
  • Depth : 從根節點到該節點的路徑長度
  • Height : 該節點到最遠葉節點的路徑長度
  • Level : 節點層次
  • Subtree : 某節點的所有子節點後代

樹的種類

  1. 二元樹 (Binary Tree)
    • 一個節點最多兩個子節點
  2. 二元搜尋樹 (Binary Search Tree, BST)
    • 左子樹所有節點值小於根節點值
    • 右子樹所有節點值大於根節點值
  3. 平衡樹 (Balanced Tree)
  4. 滿二元樹 (Full Binary Tree)
  5. 完全二元樹 (Complete Binary Tree)
  6. B 樹 (B-Tree)
  7. 堆積 (Heap)
  8. 字典樹 (Trie)

樹的遍歷(Traversal)

  1. 深度優先遍歷 (Depth-First Search, DFS)
    • 先訪問子節點，再訪問兄弟節點
    • 3 種遍歷方法
      • 前序遍歷 (Pre-order)
      • 中序遍歷 (In-order)
      • 後續遍歷 (Post-order)
  2. 廣度優先遍歷 (Breadth-First Search, BFS)
    • 階層遍歷，一層一層從左到右

相關實作(C#)

// 節點
public class TreeNode
{
    public int val;
    public TreeNode left;
    public TreeNode right;
    public TreeNode(int val = 0, TreeNode left = null, TreeNode right = null)
    {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

// 深度優先遍歷 (Depth-First Search, DFS)
// 前序遍歷
public void PreorderTraversal(TreeNode root)
{
    if (root == null) return;
    Console.Write(root.val + " ") // 訪問節點
    PreorderTraversal(root.left); // 遞迴訪問左子樹
    PreorderTraversal(root.right); // 遞迴訪問右子樹
}

// 中序遍歷
public void InorderTraversal(TreeNode root)
{
    if (root == null) return;
    InorderTraversal(root.left);    // 遞迴遍歷左子樹
    Console.Write(root.val + " ");  // 訪問節點
    InorderTraversal(root.right);   // 遞迴遍歷右子樹
}

// 後序遍歷
public void PostorderTraversal(TreeNode root)
{
    if (root == null) return;
    PostorderTraversal(root.left);   // 遞迴遍歷左子樹
    PostorderTraversal(root.right);  // 遞迴遍歷右子樹
    Console.Write(root.val + " ");   // 訪問節點
}

// 廣度優先遍歷 (Breadth-First Search, BFS)
// 階層遍歷
public void LevelOrderTraversal(TreeNode root)
{
    if (root == null) return;

    Queue<TreeNode> queue = new Queue<TreeNode>();
    queue.Enqueue(root);

    while (queue.Count > 0)
    {
        TreeNode node = queue.Dequeue();
        Console.Write(node.val + " ");

        if (node.left != null) queue.Enqueue(node.left);
        if (node.right != null) queue.Enqueue(node.right);
    }
}

滑動視窗(Sliding Window)

發表於 2025-01-20

概述

主要就是操作一個視窗的左右邊界，去檢查範圍內的數據，適合用於 Array 或是 String
可以避免重複已經檢查過的數據，優化時間複雜度，但只適用於連續數據

簡易範例

再給定的數組[2, 5, 8 ,6 ,1]中，找出連續 3 個數字最大總和

var arr = [2, 5, 8, 6, 1];
var findThreeLargest = function (arr) {
  let curSum = 0;
  // 初始視窗總和(前3個數總和)
  for (let i = 0; i < 3; i++) {
    curSum += arr[i];
  }

  let maxSum = curSum; // 最大總和
  let maxSumIndex = 0; // 最大總和的起始位置
  for (let i = 3; i < arr.length; i++) {
    curSum = curSum - arr[i - 3] + arr[i];
    // 更新最大總和的起始位置
    if (curSum > maxSum) {
      maxSum = curSum;
      maxSumIndex = i - 3;
    }
  }
  return { maxSum, maxSumIndex };
};

文字片段

發表於 2024-12-13 分類於 Web

什麼是文字片段(Text Fragments)

Text Fragments 是一種新的網頁功能
不需使用錨點標記（anchor tags）就可以讓使用者連結至特定文字
且會自動高亮顯示並滾動至該文本位置
目前支援的版本與瀏覽器

使用方法

基本結構 :

https://example.com/page.html#:~:text=[前綴文字-,]開始文字[,結束文字][,-後綴文字]

主要組成 :

  • :~: 指令代表這是文字片段
  • text= 要搜尋的文字
  • [] 選用組件

組件說明 :

  • 前綴文字 : 應出現在開始文字前面的文字
  • 開始文字 : 目標片段開頭文字
  • 結束文字 : 目標片段結尾文字
  • 後綴文字 : 應出現在結尾文字後面的文字

樣式調整可透過::target-text進行調整

::target-text {
  background-color: yellow;
}

使用範例

http://127.0.0.1:5500/index.html

http://127.0.0.1:5500/index.html#:~:text=心放地,學的有

http://127.0.0.1:5500/index.html#:~:text=如及間沒人說利驗於民物時人&text=沒才雨叫訴

MessageHub.cs

實作 Hub 類別
這邊使用單一使用者群組做簡單的連線管理
因為是暫存在記憶體，關閉後資料就消失ㄌ
如果需要持久化，要再配合 DB 去進行持久化

public class MessageHub : Hub
{
    // 紀錄用戶的連線(單一使用者群組)
    public static readonly Dictionary<string, List<string>> _userConnections = new();

    // 連線時要做的事
    public override async Task OnConnectedAsync()
    {
        var userName = Context.GetHttpContext()?.Request.Query["user"]
          ?? throw new ArgumentNullException("未知使用者");

        if (_userConnections.TryGetValue(userName,out var connectionIds))
        {
            if (!connectionIds.Contains(Context.ConnectionId))
            {
                connectionIds.Add(Context.ConnectionId);
            }
        }
        else
        {
            _userConnections.Add(userName, new List<string> { Context.ConnectionId });
        }

        await Groups.AddToGroupAsync(Context.ConnectionId, userName);
        await base.OnConnectedAsync();
    }

    // 斷線時要做的事
    public override async Task OnDisconnectedAsync(Exception? exception)
    {

        var user = _userConnections.FirstOrDefault(x => x.Value.Contains(Context.ConnectionId)).Key;

        if (user is not null)
        {
            // 移除該用戶的連線
            _userConnections[user].Remove(Context.ConnectionId);

            // 如果該用戶已經沒有任何連線
            if (!_userConnections[user].Any())
            {
                // 移除用戶
                _userConnections.Remove(user);
            }

            await Groups.RemoveFromGroupAsync(Context.ConnectionId, user);
        }

        await base.OnDisconnectedAsync(exception);
    }
}

WebhookController.cs

簡易提供全域、使用者群組、現有連線 api 端點

[Route("api/[controller]")]
[ApiController]
public class WebhookController : ControllerBase
{
    private readonly IHubContext<MessageHub> hub;

    public WebhookController(IHubContext<MessageHub> hub)
    {
        this.hub = hub;
    }

    //全域推播
    [HttpPost("all")]
    public async Task<IActionResult> SendAll([FromBody]MessageDto message)
    {
        await hub.Clients.All.SendAsync("ReceiveMessage", message);
        return Ok();
    }

    //群組推播(使用者群組)
    [HttpPost("group")]
    public async Task<IActionResult> SendGroup([FromBody] WebhookPayload payload)
    {
        var message = new MessageDto
        {
            Sender = payload.Sender,
            Content = payload.Message,
            TimeSpan = DateTime.Now
        };
        await hub.Clients.Groups(payload.Receiver).SendAsync("ReceiveGroupMessage", message);
        return Ok();
    }

    // 現有連線清單
    [HttpGet]
    public IActionResult GetUsers()
    {
        var users = MessageHub._userConnections;
        return Ok(users);
    }

    public class WebhookPayload
    {
        public string Message { get; set; }
        public string Sender { get; set; }
        public string Receiver { get; set; }
    }
}

前端使用 quasar vue typescript

<script setup lang="ts">
import * as signalR from '@microsoft/signalr';
import { ref, onMounted } from 'vue';

type MessageDto = {
  sender: string;
  content: string;
  timeSpan: Date;
};
const user = 'admin';
const receiveMsgs = ref<MessageDto[]>([]);

const connection = new signalR.HubConnectionBuilder()
  .withUrl(`https://localhost:7244/hub?user=${user}`)
  .withAutomaticReconnect()
  .build();

//監聽全頻推播
connection.on('ReceiveMessage', (res: MessageDto) => {
  receiveMsgs.value.push(res);
});

//監聽群組推播
connection.on('ReceiveGroupMessage', (res: MessageDto) => {
  receiveMsgs.value.push(res);
});

//重新連線
connection.onreconnected(() => {
  console.log('Reconnected successfully.');
});

connection.onclose(async () => {
  console.log('Disconnected. Retrying connection...');
  await reconnect();
});

const reconnect = async () => {
  let attempts = 0;
  const maxAttempts = 100; // 最大嘗試次數
  const retryDelay = 30000; // 嘗試間隔(ms)

  while (attempts < maxAttempts) {
    try {
      await connection.start();
      console.log('Reconnected successfully.');
      return; // 連線成功,退出循環
    } catch (err) {
      attempts++;
      console.log(`Reconnection attempt ${attempts} failed.`, err);
      if (attempts < maxAttempts) {
        await new Promise((resolve) => setTimeout(resolve, retryDelay)); // 等待后重试
      } else {
        console.log('Failed to reconnect after several attempts.')
      }
    }
  }
};

onMounted(async () => {
  try {
    await connection.start();
    console.log(`${user} has connected`)
  } catch (err) {
    console.log(err);
  }
});
</script>

LINQ

發表於 2024-12-11 分類於 CSharp

LINQ ( Language Integrated Query)

微軟開發的技術具有查詢資料的程式語言
讓工程師能夠使用原生 C#對資料進行處理
且相同語法可以查詢不同類型的資料，降低學習成本
擴充類型 :

  • LINQ to Objects
  • LINQ to SQL
  • LINQ to XML
  • …etc

LINQ 查詢的兩種語法

  1. 查詢表達式 ( Query Expression Syntax )
  • 類似 SQL 的語法 : 使用 select 、 from 、where 等關鍵字
  • 較容易理解
  • 結構固定 : 必須 from 起始，select 或 group by 結尾
static void Main()
{
    // Data source.
    int[] scores = new int[] { 90, 71, 82, 93, 75, 82 };

    // Query Expression.
    IEnumerable<int> scoreQuery = //query variable
        from score in scores //required
        where score > 80 // optional
        orderby score descending // optional
        select score; //must end with select or group

    // Execute the query to produce the results
    foreach (var testScore in scoreQuery)
    {
        Console.WriteLine(testScore);
    }

    // Output: 93 90 82 82
}
  1. 流利語法（Fluent Syntax）
  • 靈活性高 : 可以方便地鏈結調用多個方法
  • Lambda 表達式 : 代碼簡潔
  • 沒有固定結構 : 沒有特定的開始或結束語句
static void Main()
{
    // Data source.
    int[] scores = new int[] { 90, 71, 82, 93, 75, 82 };

    // Query Expression.
    var scoreQuery = scores
        .Where(score => score > 80)
        .OrderByDescending(score => score);

    // Execute the query to produce the results
    foreach (var testScore in scoreQuery)
    {
        Console.WriteLine(testScore);
    }

    // Output: 93 90 82 82
}
特徵 查詢表達式 流利語法
風格 類似 sql 方法鏈結
可讀性 高 中
結構要求 from 開頭、select 或 group by 結尾 無特定要求
靈活性 某些情況下較低 更高的靈活性，尤其是動態

時間複雜度

概述

時間複雜度代表的是演算法隨著資料量變大時的增長趨勢
數學符號大O記號(big-O notation)

常見型別

  • 常數階 O(1)
int Constant(int n)
{
    int count = 0;
    // 迴圈次數與 n 無關
    int size = 10000;
    for (int i = 0; i < size; i++)
    {
        count++;
    }
    return count;
}
  • 線性階 O(n)
int Linear(int n)
{
    int count = 0;
    // 迴圈次數 n 次
    for (int i = 0; i < n; i++)
    {
        count++;
    }
    return count;
}
  • 平方階 O(n2n^2n2)
int Quadratic(int n)
{
    int count = 0;
    // 迴圈次數 n*n 次
    for (int i = 0; i < n; i++)
    {
        for (int j = 0; j < n; j++)
        {
            count++;
        }
    }
    return count;
}
  • 指數階 O(2n2^n2n)
int Exponential(int n)
{
    int count = 0;
    int bas = 1;
    for (int i = 0; i < n; i++)
    {
        //迴圈次數為2^n次
        for (int j = 0; j < bas; j++)
        {
            count++;
        }
        bas *= 2;
    }
    return count;
}
  • 對數階 O(logn)
int Logarithmic(int n)
{
    int count = 0;
    while (n > 1) {
        n /= 2;
        count++;
    }
    return count;
}
  • 線性對數階 O(nlogn)
int LinearLogRecur(int n)
{
    if (n <= 1) return 1;
    int count = LinearLogRecur(n / 2);
    for (int i = 0; i < n; i++)
    {
        count++;
    }
    return count;
}
  • 階乘階 O(n!)
int FactorialRecur(int n)
{
    if (n == 0) return 1;
    int count = 0;
    // n * (n - 1) * (n - 2) * ... * 2 * 1
    for (int i = 0; i < n; i++)
    {
        count += FactorialRecur(n - 1);
    }
    return count;
}

迭代與遞迴

發表於 2024-11-27 更新於 2024-12-30 分類於 概念

迭代

迭代(iteration)是一種重複執行任務的控制結構
在一定的條件下重複執行，直到條件不再滿足

  • for
    最常見的迭代形式
int ForLoop(int n)
{
    int result = 0;
    for (int i = 1; i <= n; i--)
    {
        result += i;
    }
    return result;
}
  • while
    while 跟 for 類似，但是 while 會比較好控制條件的設定與變化
    for 的程式碼較簡潔、while 的迴圈控制較靈活
int WhileLoop(int n)
{
    int result = 0;
    int i = 1; //初始條件
    while (i <= n)
    {
        result += i;
        //更新條件
        i +=1;
        i *=2;
    }
    return result;
}
  • 巢狀
    迴圈包迴圈
string NestForLoop(int n)
{
    var result = new StringBuilder();
    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            result.Append($"({i},{j})");
        }
    }
    return result.ToString();
}

遞迴

遞迴(recursion)函式呼叫自己本身
主要三要素

  1. 終止條件 : 什麼時候結束往下層傳遞
  2. 遞迴呼叫 : 將參數往下層傳遞給自己
  3. 結果返回 : 回傳結果給上層
int Recur(int n)
{
    // 終止條件
    if (n == 1) return 1;
    // 遞迴呼叫
    int res = Recur(n - 1);
    // 返回結果
    return n + res;
}

由於遞迴每次都要呼叫重新產生本身函式，會有額外的資源開銷
所以遞迴的次數是有限的，太多記憶體會爆炸，效率也會比迭代來的低

比較

迭代 遞迴
效率 較高，沒有函式產生開銷 每次遞迴呼叫都有產生開銷
記憶體 通常固定大小 根據遞迴函式產生占用大量記憶體
適用 簡單任務、直觀、可讀佳 複雜任務分解(樹、圖、分治、回溯等)、結構明確

消耗品、治療與寶箱(重要!!!)

這個技巧適合用於你想要秒殺菁英怪，因為他會掉落一個傳奇寶箱可以回復 100HP

這是一個很有用的策略，但常常被忽略

除非你的操作很神，要不然你一定會需要補血，他會在關鍵時刻救你一命 🤡

S-Tier Items

為什麼要盡量取得這些裝備???

  • 讓你遊戲更順利
  • 巨大價值，且幾乎回本
  • 傷害越高，你殺敵的速度就越快，殺的敵人越多，你獲得的材料也就越多。
  • 越早獲得圖片中 S 級裝備裡那些提供收穫（Harvesting）的裝備，它們回本的速度就越快。

注意!!!
為什麼那些可以增加敵人數量的裝備是好東西??
因為每回合刷新的敵人越多，整場遊戲中獲得的材料也會越多。因此，記得留意這類裝備。

特別

如果你覺得你常常被菁英怪或 BOSS 貼著打或是覺得移動速度不夠躲怪那就選它

為什麼要盡量避免選擇這些物品??
因為這些裝備的副作用會隨著遊戲的進行越來越擴大
