文字片段

發表於 2024-12-13

什麼是文字片段(Text Fragments)

Text Fragments 是一種新的網頁功能
不需使用錨點標記（anchor tags）就可以讓使用者連結至特定文字
且會自動高亮顯示並滾動至該文本位置
目前支援的版本與瀏覽器

使用方法

基本結構 :

https://example.com/page.html#:~:text=[前綴文字-,]開始文字[,結束文字][,-後綴文字]

主要組成 :

  • :~: 指令代表這是文字片段
  • text= 要搜尋的文字
  • [] 選用組件

組件說明 :

  • 前綴文字 : 應出現在開始文字前面的文字
  • 開始文字 : 目標片段開頭文字
  • 結束文字 : 目標片段結尾文字
  • 後綴文字 : 應出現在結尾文字後面的文字

樣式調整可透過::target-text進行調整

::target-text {
  background-color: yellow;
}

使用範例

http://127.0.0.1:5500/index.html

http://127.0.0.1:5500/index.html#:~:text=心放地,學的有

http://127.0.0.1:5500/index.html#:~:text=如及間沒人說利驗於民物時人&text=沒才雨叫訴

MessageHub.cs

實作 Hub 類別
這邊使用單一使用者群組做簡單的連線管理
因為是暫存在記憶體，關閉後資料就消失ㄌ
如果需要持久化，要再配合 DB 去進行持久化

public class MessageHub : Hub
{
    // 紀錄用戶的連線(單一使用者群組)
    public static readonly Dictionary<string, List<string>> _userConnections = new();

    // 連線時要做的事
    public override async Task OnConnectedAsync()
    {
        var userName = Context.GetHttpContext()?.Request.Query["user"]
          ?? throw new ArgumentNullException("未知使用者");

        if (_userConnections.TryGetValue(userName,out var connectionIds))
        {
            if (!connectionIds.Contains(Context.ConnectionId))
            {
                connectionIds.Add(Context.ConnectionId);
            }
        }
        else
        {
            _userConnections.Add(userName, new List<string> { Context.ConnectionId });
        }

        await Groups.AddToGroupAsync(Context.ConnectionId, userName);
        await base.OnConnectedAsync();
    }

    // 斷線時要做的事
    public override async Task OnDisconnectedAsync(Exception? exception)
    {

        var user = _userConnections.FirstOrDefault(x => x.Value.Contains(Context.ConnectionId)).Key;

        if (user is not null)
        {
            // 移除該用戶的連線
            _userConnections[user].Remove(Context.ConnectionId);

            // 如果該用戶已經沒有任何連線
            if (!_userConnections[user].Any())
            {
                // 移除用戶
                _userConnections.Remove(user);
            }

            await Groups.RemoveFromGroupAsync(Context.ConnectionId, user);
        }

        await base.OnDisconnectedAsync(exception);
    }
}

WebhookController.cs

簡易提供全域、使用者群組、現有連線 api 端點

[Route("api/[controller]")]
[ApiController]
public class WebhookController : ControllerBase
{
    private readonly IHubContext<MessageHub> hub;

    public WebhookController(IHubContext<MessageHub> hub)
    {
        this.hub = hub;
    }

    //全域推播
    [HttpPost("all")]
    public async Task<IActionResult> SendAll([FromBody]MessageDto message)
    {
        await hub.Clients.All.SendAsync("ReceiveMessage", message);
        return Ok();
    }

    //群組推播(使用者群組)
    [HttpPost("group")]
    public async Task<IActionResult> SendGroup([FromBody] WebhookPayload payload)
    {
        var message = new MessageDto
        {
            Sender = payload.Sender,
            Content = payload.Message,
            TimeSpan = DateTime.Now
        };
        await hub.Clients.Groups(payload.Receiver).SendAsync("ReceiveGroupMessage", message);
        return Ok();
    }

    // 現有連線清單
    [HttpGet]
    public IActionResult GetUsers()
    {
        var users = MessageHub._userConnections;
        return Ok(users);
    }

    public class WebhookPayload
    {
        public string Message { get; set; }
        public string Sender { get; set; }
        public string Receiver { get; set; }
    }
}

前端接收 webhook

前端使用 quasar vue typescript

<script setup lang="ts">
import * as signalR from '@microsoft/signalr';
import { ref, onMounted } from 'vue';

type MessageDto = {
  sender: string;
  content: string;
  timeSpan: Date;
};
const user = 'admin';
const receiveMsgs = ref<MessageDto[]>([]);

const connection = new signalR.HubConnectionBuilder()
  .withUrl(`https://localhost:7244/hub?user=${user}`)
  .withAutomaticReconnect()
  .build();

//監聽全頻推播
connection.on('ReceiveMessage', (res: MessageDto) => {
  receiveMsgs.value.push(res);
});

//監聽群組推播
connection.on('ReceiveGroupMessage', (res: MessageDto) => {
  receiveMsgs.value.push(res);
});

//重新連線
connection.onreconnected(() => {
  console.log('Reconnected successfully.');
});

connection.onclose(async () => {
  console.log('Disconnected. Retrying connection...');
  await reconnect();
});

const reconnect = async () => {
  let attempts = 0;
  const maxAttempts = 100; // 最大嘗試次數
  const retryDelay = 30000; // 嘗試間隔(ms)

  while (attempts < maxAttempts) {
    try {
      await connection.start();
      console.log('Reconnected successfully.');
      return; // 連線成功,退出循環
    } catch (err) {
      attempts++;
      console.log(`Reconnection attempt ${attempts} failed.`, err);
      if (attempts < maxAttempts) {
        await new Promise((resolve) => setTimeout(resolve, retryDelay)); // 等待后重试
      } else {
        console.log('Failed to reconnect after several attempts.')
      }
    }
  }
};

onMounted(async () => {
  try {
    await connection.start();
    console.log(`${user} has connected`)
  } catch (err) {
    console.log(err);
  }
});
</script>

LINQ ( Language Integrated Query)

微軟開發的技術具有查詢資料的程式語言
讓工程師能夠使用原生 C#對資料進行處理
且相同語法可以查詢不同類型的資料，降低學習成本
擴充類型 :

  • LINQ to Objects
  • LINQ to SQL
  • LINQ to XML
  • …etc

LINQ 查詢的兩種語法

  1. 查詢表達式 ( Query Expression Syntax )
  • 類似 SQL 的語法 : 使用 select 、 from 、where 等關鍵字
  • 較容易理解
  • 結構固定 : 必須 from 起始，select 或 group by 結尾
static void Main()
{
    // Data source.
    int[] scores = new int[] { 90, 71, 82, 93, 75, 82 };

    // Query Expression.
    IEnumerable<int> scoreQuery = //query variable
        from score in scores //required
        where score > 80 // optional
        orderby score descending // optional
        select score; //must end with select or group

    // Execute the query to produce the results
    foreach (var testScore in scoreQuery)
    {
        Console.WriteLine(testScore);
    }

    // Output: 93 90 82 82
}
  1. 流利語法（Fluent Syntax）
  • 靈活性高 : 可以方便地鏈結調用多個方法
  • Lambda 表達式 : 代碼簡潔
  • 沒有固定結構 : 沒有特定的開始或結束語句
static void Main()
{
    // Data source.
    int[] scores = new int[] { 90, 71, 82, 93, 75, 82 };

    // Query Expression.
    var scoreQuery = scores
        .Where(score => score > 80)
        .OrderByDescending(score => score);

    // Execute the query to produce the results
    foreach (var testScore in scoreQuery)
    {
        Console.WriteLine(testScore);
    }

    // Output: 93 90 82 82
}
特徵 查詢表達式 流利語法
風格 類似 sql 方法鏈結
可讀性 高 中
結構要求 from 開頭、select 或 group by 結尾 無特定要求
靈活性 某些情況下較低 更高的靈活性，尤其是動態

時間複雜度

發表於 2024-11-27 分類於 演算法

概述

時間複雜度代表的是演算法隨著資料量變大時的增長趨勢
數學符號大O記號(big-O notation)

常見型別

  • 常數階 O(1)
int Constant(int n)
{
    int count = 0;
    // 迴圈次數與 n 無關
    int size = 10000;
    for (int i = 0; i < size; i++)
    {
        count++;
    }
    return count;
}
  • 線性階 O(n)
int Linear(int n)
{
    int count = 0;
    // 迴圈次數 n 次
    for (int i = 0; i < n; i++)
    {
        count++;
    }
    return count;
}
  • 平方階 O(n2n^2n2)
int Quadratic(int n)
{
    int count = 0;
    // 迴圈次數 n*n 次
    for (int i = 0; i < n; i++)
    {
        for (int j = 0; j < n; j++)
        {
            count++;
        }
    }
    return count;
}
  • 指數階 O(2n2^n2n)
int Exponential(int n)
{
    int count = 0;
    int bas = 1;
    for (int i = 0; i < n; i++)
    {
        //迴圈次數為2^n次
        for (int j = 0; j < bas; j++)
        {
            count++;
        }
        bas *= 2;
    }
    return count;
}
  • 對數階 O(logn)
int Logarithmic(int n)
{
    int count = 0;
    while (n > 1) {
        n /= 2;
        count++;
    }
    return count;
}
  • 線性對數階 O(nlogn)
int LinearLogRecur(int n)
{
    if (n <= 1) return 1;
    int count = LinearLogRecur(n / 2);
    for (int i = 0; i < n; i++)
    {
        count++;
    }
    return count;
}
  • 階乘階 O(n!)
int FactorialRecur(int n)
{
    if (n == 0) return 1;
    int count = 0;
    // n * (n - 1) * (n - 2) * ... * 2 * 1
    for (int i = 0; i < n; i++)
    {
        count += FactorialRecur(n - 1);
    }
    return count;
}

迭代與遞迴

發表於 2024-11-27 分類於 演算法

迭代

迭代(iteration)是一種重複執行任務的控制結構
在一定的條件下重複執行，直到條件不再滿足

  • for
    最常見的迭代形式
int ForLoop(int n)
{
    int result = 0;
    for (int i = 1; i <= n; i--)
    {
        result += i;
    }
    return result;
}
  • while
    while 跟 for 類似，但是 while 會比較好控制條件的設定與變化
    for 的程式碼較簡潔、while 的迴圈控制較靈活
int WhileLoop(int n)
{
    int result = 0;
    int i = 1; //初始條件
    while (i <= n)
    {
        result += i;
        //更新條件
        i +=1;
        i *=2;
    }
    return result;
}
  • 巢狀
    迴圈包迴圈
string NestForLoop(int n)
{
    var result = new StringBuilder();
    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= n; j++)
        {
            result.Append($"({i},{j})");
        }
    }
    return result.ToString();
}

遞迴

遞迴(recursion)函式呼叫自己本身
主要三要素

  1. 終止條件 : 什麼時候結束往下層傳遞
  2. 遞迴呼叫 : 將參數往下層傳遞給自己
  3. 結果返回 : 回傳結果給上層
int Recur(int n)
{
    // 終止條件
    if (n == 1) return 1;
    // 遞迴呼叫
    int res = Recur(n - 1);
    // 返回結果
    return n + res;
}

由於遞迴每次都要呼叫重新產生本身函式，會有額外的資源開銷
所以遞迴的次數是有限的，太多記憶體會爆炸，效率也會比迭代來的低

比較

迭代 遞迴
效率 較高，沒有函式產生開銷 每次遞迴呼叫都有產生開銷
記憶體 通常固定大小 根據遞迴函式產生占用大量記憶體
適用 簡單任務、直觀、可讀佳 複雜任務分解(樹、圖、分治、回溯等)、結構明確

C# 資料快取

發表於 2024-11-20 分類於 CSharp

概述

有時候一些大型資料且異動少的資料
每次使用就要從資料重新撈取就很浪費效能資源
這個時候就是快取出場的時候

Install

dotnet add package Microsoft.Extensions.Caching.Memory

Extensions.cs

public static void AddExtension(this IServiceCollection services)
{
    services.AddMemoryCache();
    // services.AddScoped<CacheHelper>(); //如果有使用整合工具類
}

Usage

[Route("api/[controller]")]
 [ApiController]
 public class DataController : ControllerBase
 {
     private readonly DataBaseService dataBaseService;
     private readonly IMemoryCache memoryCache;

     public DataController(DataBaseService dataBaseService, IMemoryCache memoryCache)
     {
         this.memoryCache = memoryCache;
         this.dataBaseService = dataBaseService;
     }

     [HttpGet("/search")]
     public async Task<IActionResult> Search(string keyword)
     {
        // 從記憶體快取查看是否存在，沒再去資料庫撈資料並儲存快取
         if (memoryCache.TryGetValue("MainDataCache", out List<Prs006> dataList))
         {
             var result = dataList?
                 .Where(data => data.Ordnme?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true
                                || data.Scitn?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true
                                || data.Keyword?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true)
                 .ToList() ?? new List<Prs006>();

             return Ok(result);
         }
         else
         {
             var dbData = await dataBaseService.GetDatas();
             //寫入記憶體儲存，一天後自動過期
             memoryCache.Set("MainDataCache", dbData, TimeSpan.FromDays(1));

             var result = dbData?
                 .Where(data => data.Ordnme?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true
                                || data.Scitn?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true
                                || data.Keyword?.Contains(keyword, StringComparison.OrdinalIgnoreCase) == true)
                 .ToList() ?? new List<Prs006>();

             return Ok(result);
         }
     }

CacheHelper.cs(Optional)

public class CacheHelper
{
    private readonly IMemoryCache cache;

    public CacheHelper(IMemoryCache cache)
    {
        this.cache = cache;
    }

    public void Set<T>(string key, T value, int expire = 300)
    {
        var options = new MemoryCacheEntryOptions
        {
            AbsoluteExpirationRelativeToNow = TimeSpan.FromSeconds(expire),
        };
        cache.Set(key, value, options);
    }

    public T? Get<T>(string key)
    {
        cache.TryGetValue<T>(key, out var value);
        return value;
    }

    public void Remove(string key)
    {
        cache.Remove(key);
    }

    public void Update<T>(string key, T value, int expire)
    {
        cache.Remove(key);
        Set<T>(key, value, expire);
    }

    public bool Contains(string key)
    {
        return cache.TryGetValue(key, out _);
    }
}

CurlHelper.cs

public class CurlHelper
    {
        // curl.exe 路徑
        private static readonly string CurlPath = $"C:\\AP06BZ\\EXE\\CURL.EXE";

        /// <summary>
        /// Get 請求
        /// </summary>
        /// <param name="url"></param>
        /// <param name="headers"></param>
        /// <returns></returns>
        public static string GetRequest(string url, Dictionary<string, string> headers = null)
        {
            string arugments = $"-X GET \"{url}\"";
            if (headers != null)
            {
                foreach (var header in headers)
                {
                    arugments += $" -H \"{header.Key}: {header.Value}\"";
                }
            }
            return ExecuteCurlCommand(arugments);
        }

        /// <summary>
        /// POST 請求(JSON資料)
        /// </summary>
        /// <param name="url"></param>
        /// <param name="jsonData"></param>
        /// <param name="headers"></param>
        /// <returns></returns>
        public static string PostRequest(string url, string jsonData, Dictionary<string, string> headers = null)
        {
            string arguments = $"-X POST -H \"Content-Type: application/json\" -d \"{jsonData}\" \"{url}\"";
            if (headers != null)
            {
                foreach (var header in headers)
                {
                    arguments += $" -H \"{header.Key}: {header.Value}\"";
                }
            }
            return ExecuteCurlCommand(arguments);
        }

        /// <summary>
        /// POST 請求(表單資料)
        /// </summary>
        /// <param name="url"></param>
        /// <param name="formData"></param>
        /// <param name="headers"></param>
        /// <returns></returns>
        public static string PostFormRequest(string url, Dictionary<string, string> formData, Dictionary<string, string> headers = null)
        {
            string arguments = "-X POST";
            foreach (var item in formData)
            {
                arguments += $" -F \"{item.Key}={item.Value}\"";
            }
            arguments += $" \"{url}\"";
            if (headers != null)
            {
                foreach (var header in headers)
                {
                    arguments += $" -H \"{header.Key}: {header.Value}\"";
                }
            }
            return ExecuteCurlCommand(arguments);
        }


        /// <summary>
        /// 執行curl命令返回輸出
        /// </summary>
        /// <param name="arguments"></param>
        /// <returns></returns>
        private static string ExecuteCurlCommand(string arguments)
        {
            var startInfo = new ProcessStartInfo
            {
                FileName = CurlPath,
                Arguments = arguments,
                RedirectStandardOutput = true,
                UseShellExecute = false,
                CreateNoWindow = true,
            };
            using (var p = Process.Start(startInfo))
            {
                using (var reader = p.StandardOutput)
                {
                    return reader.ReadToEnd();
                }
            }
        }
    }

C# 使用IdGen產生Id(雪花算法)

發表於 2024-11-19 分類於 CSharp

雪花算法

概述

公司的資料表常常使用很奇怪的欄位來去當識別鍵(例如時間戳)，既然都用時間戳了不如用雪花算法產 ID 當 PK
雪花算法是 Twitter 開源的生成分散式全域產生 ID 的演算法

格式

雪花 ID 有 64 位元

  • 1 位為符號位始終是 0
  • 41 位為時間戳，精度毫秒
  • 10 位可為(10 位機器 ID) 或 (5 位數據中心 ID + 5 位機器 ID)，可識別 1024 台機器
  • 12 位為序列號 => 代表每毫秒可以有 4096 個 ID

優缺點

優點

  1. ID 包含時間戳，有序可排序
  2. 簡單速度快

缺點

  1. 依賴系統時間，如果系統時間回退或跳躍，會產生不唯一 ID
  2. 自己手動分配機器 ID，分配不當，產生不唯一 ID

Install

dotnet add package IdGen

DI 注入(Optionl)

dotnet add package IdGen.DependencyInjection

Usage

一般

var generator = new IdGenerator(0);
var id = generator.CreateId();

依賴注入
